1) Рассмотрю организацию классов потоков на примере строковых потоковых классов. 
	- Есть шаблонные классы basic_istringsream<>, basic_ostringsream<>, basic_stringstream<> для ввода и вывода данных. У каждого из них по две специализации для работы со string и wstring.
	- Есть шаблонный класс basic_stringbuf<> с двумя аналогичными специализациями. Используется другими строковыми потоковыми классами для фактического ввода и вывода символов.
2) Реализованы флаги состояния, свойства локали потока, ширина следующего форматированного элемента.
3) Манипуляторы, грубо говоря, применяются в моменте, а флаги можно зафиксировать на поток, даже сохранить в отдельную переменную. Флаги - описывают состояние потока в целом, а манипуляторы нет.
4) Директорий, разделителей и корневой директории (для нее существует отдельный символ).
5) Для удобного представления и манипулирования данными в форме, понятной пользователю.
